# Graph DB Tool

Graph DB 관리 도구 - 스키마 관리 및 메타데이터 편집 기능

## 주요 기능

1. **스키마 관리 시스템**
   - 노드 타입별 프로퍼티 형식 정의 및 표시 UI
   - Story, Back, Chat 등 각 노드 타입의 프로퍼티 형식 관리
   - 스키마 변경 시 모든 파트에 자동 반영
   - 프로퍼티 추가/수정/삭제 기능

2. **그래프 업로드 시스템**
   - JSON 파일 업로드
   - 업로드된 프로퍼티를 스키마 형식에 자동 매핑
   - 프로퍼티 매핑 수동 조정 가능
   - 스키마 검증 및 형식 변환

3. **메타데이터 편집 도구**
   - 쿼리 없이 제목, 유니버스, 설명 등 메타데이터 수정
   - 간단한 UI로 안전하게 메타데이터 관리
   - 유니버스 삭제 기능

## 문제 해결

### 기존 문제점
1. Graph 형식 변경 시 Story, Back, Chat 모든 파트 수정 필요
   - **해결**: 중앙화된 스키마 관리 시스템으로 한 곳에서 수정하면 모든 파트에 반영

2. 메타데이터 수정을 위해 쿼리 작성 필요
   - **해결**: UI 기반 메타데이터 편집 도구로 쿼리 없이 안전하게 수정

## 프로젝트 구조

```
src/
├── components/
│   ├── SchemaManager/      # 스키마 관리 컴포넌트
│   ├── GraphUploader/      # 그래프 업로드 컴포넌트
│   └── MetadataEditor/     # 메타데이터 편집 컴포넌트
├── types/
│   ├── schema.ts           # 스키마 타입 정의
│   └── graph.ts            # 그래프 타입 정의
├── services/
│   └── api.ts              # API 호출 서비스 (Neo4j 연동 준비)
├── App.tsx                 # 메인 앱 컴포넌트
└── main.tsx                # 진입점
```

## 설치 및 실행

### 프론트엔드
```bash
npm install
npm run dev
```

서버는 `http://localhost:3000`에서 실행됩니다.

## Neo4j 연동

현재는 프론트엔드만 구현되어 있으며, `src/services/api.ts`에서 API 호출 구조가 정의되어 있습니다.
실제 Neo4j Graph DB와 연동하려면 백엔드 API를 구현하고 해당 파일의 TODO 주석 부분을 실제 API 호출로 변경하면 됩니다.

## 사용 방법

1. **스키마 관리**: 각 노드 타입(Story, Back, Chat)의 프로퍼티 형식을 정의하고 관리합니다.
2. **그래프 업로드**: JSON 파일을 업로드하고 스키마에 맞게 프로퍼티를 매핑합니다.
3. **메타데이터 편집**: ID를 입력하여 메타데이터를 불러온 후 편집할 수 있습니다.
